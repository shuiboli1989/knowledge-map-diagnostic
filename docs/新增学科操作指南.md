# 新增学科操作指南

## 概述

本文档记录了向教育诊断原型系统中新增一门学科的完整流程，以"哲学通论"的实际操作为参考。

---

## 前置条件

- 学科的知识图谱 Excel 文件（`.xlsx`），放在 `data/` 目录下
- Excel 中需包含：知识点层级、知识点名称、知识点ID、父级知识点ID、同级排序等字段
- Python 环境已安装 `pandas`、`openpyxl`、`requests` 等依赖

---

## 操作步骤

### 第一步：确定课程ID

按照项目命名规范，确定一个 `course_id`，格式为 `course_xxx`。例如：

| 课程 | course_id |
|------|-----------|
| 金融学基础 | course_finance_101 |
| 线性代数 | course_linear_algebra |
| 中国金融学 | course_china_finance |
| 哲学通论 | course_philosophy |

### 第二步：生成知识图谱 JSON

1. 将 Excel 文件放入 `data/` 目录
2. 用 pandas 读取 Excel，提取层级3的知识点作为图谱节点
3. 生成 `data/graph_{course_id}.json`，格式如下：

```json
{
  "nodes": [
    {
      "id": "node_xxx_001",
      "name": "知识点名称",
      "prerequisites": ["node_xxx_000"],
      "description": {
        "core_concept": "核心概念描述",
        "testable_angles": {
          "memory": "记忆层考察角度",
          "understanding": "理解层考察角度",
          "application": "应用层考察角度",
          "analysis": "分析层考察角度"
        },
        "confusable_points": "易混淆点描述"
      },
      "difficulty_coeff": 0.5
    }
  ]
}
```

关键注意事项：
- `id` 格式：`node_{学科缩写}_{三位数序号}`
- `prerequisites`：根据知识点间的实际逻辑关系填写，不仅限于 Excel 中的父子关系
- `difficulty_coeff`：范围 0~1，入门概念低（0.3~0.4），综合性知识点高（0.6~0.7）
- `description` 中的四个 `testable_angles` 字段必须完整
- JSON 字符串值中如果包含双引号，必须转义为 `\"`

### 第三步：生成试题

运行通用化的 LLM 试题生成脚本：

```bash
python generate_questions_llm.py {course_id}
```

例如：
```bash
python generate_questions_llm.py course_philosophy
```

如果是新学科，需要先在 `generate_questions_llm.py` 的 `COURSE_CONFIG` 字典中添加配置：

```python
COURSE_CONFIG = {
    "course_xxx": ("显示名称", "学科领域", "system prompt", "题目ID前缀"),
}
```

生成结果保存为 `data/questions_{course_id}.json`，每个知识点3道题（memory/understanding/application）。

注意事项：
- API 可能对某些内容触发敏感词过滤，需要手动补充失败的题目
- 41个节点大约需要3~5分钟
- 脚本支持断点续传（如果使用分批脚本 `gen_phil_questions.py` 的模式）

### 第四步：注册课程

在 `app.py` 的 `courses` 字典中添加一行：

```python
courses = {
    # ... 已有课程
    "course_xxx": "课程显示名称"
}
```

位置在 `app.py` 约第224行。

### 第五步：验证

1. 验证数据文件完整性：
```bash
python validate_json.py
```

2. 验证图谱和题库的一致性（所有节点都有题目，所有题目都指向有效节点）：
```python
import json
with open('data/graph_{course_id}.json') as f:
    g = json.load(f)
with open('data/questions_{course_id}.json') as f:
    q = json.load(f)
graph_ids = {n['id'] for n in g['nodes']}
q_node_ids = {qq['node_id'] for qq in q}
assert q_node_ids.issubset(graph_ids), "题目引用了不存在的节点"
assert graph_ids == q_node_ids, "有节点没有对应的题目"
```

3. 启动应用验证页面：
```bash
streamlit run app.py --server.port 8504
```

在侧边栏选择新课程，确认：
- 知识图谱可视化正常显示
- 能正常出题和答题
- 知识点掌握概率正常更新

## 文件清单

新增一门学科需要创建/修改的文件：

| 文件 | 操作 | 说明 |
|------|------|------|
| `data/{学科}.xlsx` | 新增 | 原始 Excel 知识图谱 |
| `data/graph_{course_id}.json` | 新增 | 知识图谱 JSON |
| `data/questions_{course_id}.json` | 新增 | 试题库 JSON |

## 优化空间

### 1. 图谱生成自动化

当前状态：从 Excel 到 graph JSON 的转换是手动完成的，需要人工编写每个节点的 `description`（core_concept、testable_angles、confusable_points）。

优化方向：
- 编写一个 `generate_graph_from_excel.py` 脚本，自动读取 Excel 并调用 LLM 生成节点描述
- 输入：Excel 文件路径 + course_id
- 输出：完整的 graph JSON 文件
- 前驱关系可以先从 Excel 的父子关系推导，再由 LLM 补充跨模块的逻辑依赖

### 2. 试题生成的健壮性

当前状态：LLM API 可能因敏感词过滤拒绝生成，答案分布不均匀。

优化方向：
- 失败自动重试时调整 prompt 措辞（去掉可能触发过滤的关键词）
- 增加答案分布均衡性检查，不均匀时对特定答案的题目重新生成
- 增加题目质量校验（选项格式、答案有效性、解析完整性）

### 3. 课程注册自动化

当前状态：需要手动修改 `app.py` 和 `generate_questions_llm.py` 两个文件。

优化方向：
- `app.py` 改为自动扫描 `data/graph_course_*.json` 文件来发现可用课程
- 课程元信息（显示名称、学科领域等）存入 graph JSON 的顶层字段
- 这样新增课程只需要放入数据文件，无需改代码

### 4. 一键新增学科脚本

将上述所有步骤整合为一个入口脚本：

```bash
python add_course.py --excel data/新学科.xlsx --course_id course_xxx --name "课程名称"
```

自动完成：Excel 解析 → 图谱生成 → 试题生成 → 课程注册 → 数据验证

### 5. 试题质量提升

- 增加 analysis 层级的题目（当前只有 memory/understanding/application 三层）
- 支持多选题、判断题等更多题型
- 引入题目难度标定机制，根据学生作答数据动态调整
