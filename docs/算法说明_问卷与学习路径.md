# 算法说明：入学问卷与学习路径推荐

## 概述

系统在学生首次进入课程时，通过入学问卷快速了解学生背景，调整初始掌握概率（冷启动优化）；随后基于知识图谱的拓扑结构生成个性化学习路径，引导学生按序学习。

整体流程：

```
学生进入课程 → 初始化掌握概率 → 入学问卷（自评+锚点验证）
→ 调整掌握概率 → 生成学习路径 → 逐节点答题学习 → 掌握后前进
```

---

## 一、入学问卷系统

### 1.1 设计目标

解决冷启动问题：新学生进入课程时，所有知识点的掌握概率仅由图谱结构决定，无法反映学生的实际基础。通过问卷快速获取粗粒度的背景信息，调整初始概率。

### 1.2 前置条件

问卷功能依赖知识图谱中的 `modules` 字段。没有 `modules` 的课程自动跳过问卷，向后兼容。

### 1.3 模块数据结构

在 `graph_{course_id}.json` 顶层定义模块分组：

```json
{
  "modules": [
    {
      "id": "mod_phil_01",
      "name": "哲学的本质与爱智",
      "node_ids": ["node_phil_001", "node_phil_002", "node_phil_003"],
      "representative_node_id": "node_phil_001"
    }
  ],
  "nodes": [...]
}
```

- `node_ids`：该模块包含的知识点ID列表
- `representative_node_id`：锚点测试时使用的代表性节点（从该节点的题库中选题）

### 1.4 问卷流程

#### 第一阶段：模块自评

学生对每个模块选择熟悉度等级（4级）：

| 等级 | 标签 | 乘数 | 含义 |
|------|------|------|------|
| 0 | 没接触过 | ×0.6 | 下调该模块所有节点的掌握概率 |
| 1 | 听说过 | ×0.8 | 略微下调 |
| 2 | 学过且有基础 | ×1.2 | 略微上调 |
| 3 | 掌握较好 | ×1.5 | 大幅上调 |

调整公式：

```
P_new = clamp(P_old × multiplier, 0.01, 0.94)
```

上限设为 0.94（= MASTERY_THRESHOLD - 0.01），确保问卷不会直接将节点标记为"已掌握"，必须通过答题验证。

#### 第二阶段：锚点测试

对自评等级 >= 2（"学过且有基础"及以上）的模块，系统从该模块的代表性节点题库中选一道题进行验证：

- **答对**：确认自评结果，概率不变
- **答错**：该模块所有节点的掌握概率乘以衰减系数 0.5

```
答错时：P_new = max(0.01, P_old × 0.5)
```

### 1.5 数据持久化

问卷结果保存在学生状态的 `survey_results` 字段中：

```json
{
  "survey_results": {
    "completed": true,
    "responses": {"mod_phil_01": 3, "mod_phil_02": 0},
    "anchor_tests": {"mod_phil_01": {"is_correct": true}}
  }
}
```

代码位置：`core/survey.py`

## 二、学习路径生成

### 2.1 设计目标

根据知识图谱的先修关系，为学生生成一条有序的学习路径，确保学习顺序符合知识依赖关系，跳过已掌握的节点。

### 2.2 拓扑排序（Kahn 算法）

所有路径生成的基础是对知识图谱进行拓扑排序：

```
1. 计算每个节点的入度（被多少节点依赖）
2. 将入度为 0 的节点加入队列
3. BFS：取出队首节点加入结果，将其后继节点入度减 1
4. 入度变为 0 的后继节点加入队列
5. 重复直到队列为空
```

结果：一个满足先修关系的节点ID有序列表。

### 2.3 全图谱通关模式

```
路径 = 拓扑排序结果 → 过滤掉 P(mastery) >= 0.95 的节点
```

生成包含所有未掌握节点的完整学习路径。

### 2.4 指定目标模块模式

```
1. 从目标节点出发，BFS 反向收集所有前置依赖节点
2. 对收集到的节点集合进行拓扑排序
3. 过滤掉 P(mastery) >= 0.95 的节点
```

只包含达成目标所需的最小节点集合。

代码位置：`core/learning_path.py`

## 三、学习路径可视化

### 3.1 实现方式

学习路径在"学习路径"Tab 中以 pyvis 知识图谱的形式可视化展示，复用知识图谱的拓扑结构，通过颜色、大小和标记区分不同状态的节点。

### 3.2 节点视觉编码

| 状态 | 颜色 | 大小 | 标记 | 说明 |
|------|------|------|------|------|
| 当前学习节点 | 红色 #D42B2B | 40 | ▶ + 编号 | 路径中正在学习的节点 |
| 待学习节点 | 橙色 #FF9500 | 35 | 编号 | 路径中尚未学习的节点 |
| 已掌握（路径内） | 绿色 #15803D | 35 | ✓ + 编号 | 路径中已达到掌握阈值的节点 |
| 已掌握（路径外） | 绿色 #15803D | 28 | ✓ | 不在路径中但已掌握的节点 |
| 非路径未掌握 | 灰色 #E8E8E8 | 22 | 无 | 不在学习路径中的节点 |

### 3.3 边的视觉编码

- 路径内节点之间的边：深灰色 #888888，宽度 2
- 其他边：浅灰色 #E0E0E0，宽度 1

### 3.4 图例

图谱下方显示颜色图例：当前学习（红）、待学习（橙）、已掌握（绿）、不在路径中（灰）。

## 四、学习路径答题流程

### 4.1 流程

```
生成路径 → advance_path 跳过已掌握起始节点
→ 当前节点出题 → 学生答题 → BKT 更新概率
→ 达到阈值？→ 是：advance_path 前进到下一未掌握节点
             → 否：继续出题
→ 题目用完但未达标？→ 提供"重新练习"或"跳到下一个"选择
→ 所有路径节点完成 → 显示完成信息
```

### 4.2 节点前进逻辑（advance_path）

当前节点掌握后，自动跳过后续所有已掌握的节点，定位到下一个未掌握的节点：

```python
while idx < len(path_nodes):
    if node_states[path_nodes[idx]].p_mastery < 0.95:
        break
    idx += 1
```

### 4.3 题目用完未达标处理

当某个知识点的所有题目都已做过，但掌握概率仍未达到 0.95 时，提供两个选择：

- **重新练习本知识点**：清除该节点的答题记录（仅该节点，不影响其他节点），所有题目重新可用。掌握概率保持当前值不重置，重新练习时起点更高。
- **跳到下一个知识点**：强制前进到路径中的下一个未掌握节点。

### 4.4 数据持久化

学习路径状态保存在学生状态的 `learning_path` 字段中：

```json
{
  "learning_path": {
    "mode": "full",
    "target_modules": [],
    "path_nodes": ["node_phil_004", "node_phil_005", "..."],
    "current_index": 3
  }
}
```

## 六、与其他模块的关系

| 模块 | 关系 |
|------|------|
| BKT（core/bkt.py） | 学习路径中的答题使用相同的 BKT 更新逻辑 |
| CAT（core/cat.py） | 共享 MASTERY_THRESHOLD=0.95；学习路径按拓扑序出题，CAT 按信息增益出题 |
| 初始化（core/initializer.py） | 问卷在初始化之后执行，在初始概率基础上乘以调整系数 |
| IO（utils/io.py） | 负责 survey_results 和 learning_path 的读写持久化 |

诊断模式和学习路径模式共享同一份 BKT 状态（node_states 和 answer_history），两个模式的答题结果互通。
