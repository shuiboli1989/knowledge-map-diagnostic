# 算法说明：BKT 概率诊断与 CAT 自适应选题

## 概述

系统采用标准四参数贝叶斯知识追踪（BKT）模型进行知识点掌握概率的动态估计，结合计算机化自适应测试（CAT）策略进行智能选题。整体流程为：

```
初始化掌握概率 → CAT选题 → 学生答题 → BKT更新概率 → 判断是否掌握 → 继续/结束
```

---

## 一、初始掌握概率计算

每个学生首次进入课程时，系统根据知识图谱结构为每个知识点计算初始掌握概率。

### 计算公式

```
P(L₀) = 0.6 × 基础概率 + 0.4 × 难度系数
```

### 基础概率规则

基础概率由知识点的先修节点数量决定，先修越多说明知识点越高阶，初始掌握概率越低：

| 先修节点数 | 基础概率 | 含义 |
|-----------|---------|------|
| 0 | 0.50 | 入门知识点，无前置依赖 |
| 1 | 0.35 | 有一个前置知识点 |
| 2 | 0.25 | 有两个前置知识点 |
| 3+ | 0.15 | 综合性知识点，依赖多个前置 |

### 难度系数

每个知识点在知识图谱中标注了难度系数（`difficulty_coeff`），范围 0~1：
- 0.3~0.4：入门概念
- 0.5：中等难度
- 0.6~0.7：综合性/高阶知识点

### 示例

一个有2个先修节点、难度系数0.6的知识点：
```
P(L₀) = 0.6 × 0.25 + 0.4 × 0.6 = 0.15 + 0.24 = 0.39
```

代码位置：`core/initializer.py`

## 二、BKT 贝叶斯知识追踪

### 标准四参数模型

系统采用标准 BKT 四参数模型：

| 参数 | 符号 | 默认值 | 含义 |
|------|------|--------|------|
| 初始掌握概率 | P(L₀) | 由初始化计算 | 学生首次接触知识点时的掌握概率 |
| 学习转移概率 | P(T) | 0.10 | 每次练习后，从"未掌握"转为"掌握"的概率 |
| 失误率 | P(S) | 0.15 | 已掌握但答错的概率（手滑、粗心） |
| 猜测率 | P(G) | 0.25 | 未掌握但答对的概率（四选一随机猜中） |

### 更新流程

每次答题后分两步更新掌握概率：

#### 步骤1：贝叶斯后验更新

根据答题结果（对/错）修正掌握概率的估计。

答对时：
```
P(Lₙ|correct) = P(correct|mastered) × P(Lₙ) / P(correct)

其中：
P(correct) = (1-P(S)) × P(Lₙ) + P(G) × (1-P(Lₙ))
           = 0.85 × P(Lₙ) + 0.25 × (1-P(Lₙ))
```

答错时：
```
P(Lₙ|wrong) = P(wrong|mastered) × P(Lₙ) / P(wrong)

其中：
P(wrong) = 1 - P(correct)
P(wrong|mastered) = P(S) = 0.15
```

#### 步骤2：学习转移

练习本身具有学习效果，即使答错了，做题过程也会促进学习：

```
P(Lₙ₊₁) = P(Lₙ|答题结果) + (1 - P(Lₙ|答题结果)) × P(T)
```

这意味着：
- 答对后概率大幅上升（后验上升 + 学习转移叠加）
- 答错后概率下降，但不会断崖式下跌（学习转移提供兜底）
- 持续练习即使答错，概率也会逐步回升

### 概率边界

- 下限：0.01（防止概率归零后无法恢复）
- 上限：1.0（允许自然趋近完全掌握）

### 数值示例

假设初始概率 P(L₀) = 0.50，连续答对的概率演化：

| 轮次 | 后验更新 | 学习转移后 | 界面显示 |
|------|---------|-----------|---------|
| 初始 | — | 0.5000 | 50% |
| 答对1次 | 0.7727 | 0.7955 | 80% |
| 答对2次 | 0.9432 | 0.9489 | 95% |
| 答对3次 | 0.9876 | 0.9888 | 99% |
| 答对4次 | 0.9978 | 0.9980 | 100% |

假设初始概率 P(L₀) = 0.50，连续答错的概率演化：

| 轮次 | 后验更新 | 学习转移后 | 界面显示 |
|------|---------|-----------|---------|
| 初始 | — | 0.5000 | 50% |
| 答错1次 | 0.1667 | 0.2500 | 25% |
| 答错2次 | 0.0556 | 0.1500 | 15% |
| 答错3次 | 0.0192 | 0.1173 | 12% |

代码位置：`core/bkt.py`

## 三、CAT 自适应选题

### 选题目标

在每一轮诊断中，选择"信息增益最大"的知识点出题，以最少的题目数量最准确地评估学生的知识状态。

### 节点筛选

一个知识点在满足以下任一条件时退出候选池：
1. 掌握概率 >= 0.95（已掌握，无需继续测试）
2. 该知识点的所有题目都已做过（无可用题目）

### 选题优先级

从候选池中选择节点的规则：

1. 优先选择掌握概率最接近 0.5 的节点
   - 信息增益 = |0.5 - P(mastery)|
   - 概率越接近 0.5，不确定性越高，出题的诊断价值越大
   
2. 当多个节点的不确定性相近（差值 < 0.05）时，优先选择拥有更多未掌握后继节点的节点
   - 先测试"枢纽"知识点，可以更高效地推断下游知识点的状态

### 题目选择

确定知识点后，从该知识点的题库中随机选择一道学生未做过的题目。

### 终止条件

当候选池为空时（所有知识点都已掌握或无题可出），诊断结束。

代码位置：`core/cat.py`

## 四、整体流程图

```
┌─────────────────────────────────────────────────┐
│                  学生进入课程                      │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  初始化：根据知识图谱计算每个节点的 P(L₀)          │
│  公式：P(L₀) = 0.6×基础概率 + 0.4×难度系数        │
└──────────────────────┬──────────────────────────┘
                       ▼
┌─────────────────────────────────────────────────┐
│  CAT 选题：从候选池中选择信息增益最大的节点         │
│  候选条件：P(mastery) < 0.95 且有剩余题目          │
│  优先级：不确定性最高 → 后继节点最多               │
└──────────────────────┬──────────────────────────┘
                       ▼
              ┌────────────────┐
              │  候选池为空？    │
              └───┬────────┬───┘
                  │是      │否
                  ▼        ▼
          ┌──────────┐  ┌──────────────────────┐
          │ 诊断结束  │  │ 随机选题，学生作答     │
          └──────────┘  └──────────┬───────────┘
                                   ▼
                    ┌──────────────────────────┐
                    │  BKT 更新掌握概率          │
                    │  1. 贝叶斯后验更新         │
                    │  2. 学习转移叠加           │
                    └──────────────┬───────────┘
                                   ▼
                          返回 CAT 选题
```

## 五、参数调优指南

| 参数 | 当前值 | 调大效果 | 调小效果 |
|------|--------|---------|---------|
| P(T) 学习转移 | 0.10 | 概率上升更快，更容易达到掌握 | 概率上升更慢，需要更多答对才能掌握 |
| P(S) 失误率 | 0.15 | 答错对概率的惩罚更小 | 答错惩罚更大 |
| P(G) 猜测率 | 0.25 | 答对对概率的提升更小 | 答对提升更大 |
| 掌握阈值 | 0.95 | 更难达到"已掌握"，需要更多题目 | 更容易达到"已掌握" |

这些参数目前为全局默认值。未来可以按学科或知识点粒度进行差异化配置。
